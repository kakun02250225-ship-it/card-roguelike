## 深淵の塔 - Abyss Deck -

HTML / CSS / JavaScript だけで動く、デッキ構築型ローグライトカードゲームです。
敵を倒して**ガチャコイン**を集め、**カードガチャ**でデッキを強化しながら、
**マップを選んで塔を登っていく**独自システムを搭載しています。
外部ライブラリ・画像素材は一切使わず、キャラクターは**コードで描いたSVGイラスト**です。
ブラウザで `index.html` を開くだけで遊べます。

## 遊び方（起動方法）

1. このフォルダの `index.html` をダブルクリックする（または ブラウザにドラッグ＆ドロップ）
2. タイトル画面の「⚔️ 冒険を始める」を押す

それだけです。サーバーもインストールも不要です。

## ゲームのルール

- プレイヤーには **HP（75）/ ブロック / エナジー（毎ターン3）** があります
- 毎ターン、山札から **5枚ドロー** します。山札が尽きたら捨て札をシャッフルして山札に戻します
- カードをクリックすると使用し、左上の数字ぶんエナジーを消費します
- **ブロック**は敵の攻撃を肩代わりしますが、自分のターン開始時に消えます
- 「ターン終了」を押すと手札を全部捨て、敵が行動します
- 敵の頭上には **次の行動の予告** が表示されます
  - `⚔️ 数字` … その数値の攻撃 / `🛡️ 防御` … ブロックを得る / `💢 強化` … 攻撃力アップ
- **筋力**で攻撃ダメージが増えます（その戦闘中のみ有効）
- 敵を倒すと **3枚のカードから1枚** 選んでデッキに追加でき、HPが8回復します
- 塔は **全8層**。最上階の「深淵の魔王」を倒せばクリア、HPが0になるとゲームオーバー
- **ESCキー**でいつでもメニューを開けます（続ける／最初からやり直す／タイトルへ）

## マップシステム

敵を倒す（またはマスを終える）と、**次の階のマスを2〜3択から選びます**。

| マス | 内容 |
|---|---|
| ⚔️ 戦闘 | 通常の魔物と戦う（コイン 🪙5〜12） |
| 💀 強敵 | 手強いがコインが多い（🪙15〜25）。3層目以降に出現 |
| 🔥 休憩所 | 「休む（HP25回復）」か「鍛錬（最大HP+6）」を選べる。ボス直前の階には必ず出る |
| ❓ イベント | 宝箱・癒しの泉・血の祭壇・商人……何が起こるかわからない |
| 👹 魔王の間 | 最上階（8層）。深淵の魔王（コイン🪙40）が待つ |

敵のHPは階が深くなるほど少しずつ増えていきます。

## ガチャシステム（独自要素）

- 敵を倒すと **ガチャコイン** 🪙 をドロップします
  - 通常敵：5〜12枚 ／ 強敵：15〜25枚 ／ ボス：40枚（イベントでも増減あり）
- 所持コインは画面右上に常時表示されます
- 戦闘勝利後の報酬画面に、カード報酬とは**別枠**で「🎰 ガチャを引く」ボタンが出ます
- ガチャは **1回 🪙10** 消費。コインが足りないとボタンは押せません
- 引いたカードはランダムで、**レアリティ**があります

| レアリティ | 排出率 | 特徴 |
|---|---|---|
| COMMON | 50% | 基本カード |
| RARE | 30% | 一段強いカード |
| EPIC | 15% | **ガチャ限定**の強力カード |
| LEGENDARY | 5% | **ガチャ限定**の最強カード |

- ガチャ結果画面では、カード名・レアリティ・効果がレアリティ色の演出付きで表示されます
- 課金・現金要素は一切ありません。コインはゲーム内ドロップだけで完結します

## ファイル構成

```
card-roguelike/
├── index.html   … 画面の骨組み（戦闘画面・タイトル・報酬・結果のオーバーレイ）
├── style.css    … ダークファンタジー風の見た目・カードや演出のスタイル
├── script.js    … ゲームロジック本体
└── README.md    … このファイル
```

### script.js の中身（読む順番のガイド）

1. **カードデータ（CARD_LIBRARY）** … カードは「効果の組み合わせ＋レアリティ」をデータで持つだけ。新カードはここに1個足せば追加できる
2. **ガチャ設定（GACHA_COST / RARITY_TABLE）** … コスト・排出率はここの数値を変えるだけで調整できる
3. **SVGアート（ART）** … キャラクターのイラストはコードで描いたSVG。画像ファイル不要
4. **敵データ（NORMAL_ENEMIES / ELITE_ENEMIES / BOSS_ENEMY）** … 行動パターン（pattern）とコインドロップ量（coinMin / coinMax）を持つ
5. **マップ・イベント（NODE_INFO / EVENTS）** … マスの種類とイベントの選択肢・効果をデータで定義
6. **ゲーム状態（state）** … HP・山札・手札・捨て札・所持コイン・現在の階など、ゲームの「今」を全部この1つのオブジェクトで管理
7. **マップ進行** … `showMap()`（道の選択）→ `enterNode()`（戦闘／休憩／イベントへ分岐）
8. **戦闘の流れ** … `startBattle → startPlayerTurn → playCard → endTurn → enemyAct → …` の繰り返し
9. **ガチャ** … `rollGachaRarity()`（重み付き抽選）→ `pullGacha()`（カード抽選とデッキ追加）→ `showGachaResult()`（結果表示）
10. **演出** … 攻撃の踏み込み（lunge）、斬撃エフェクト、ダメージ数字のポップアップはCSSアニメーション
11. **描画** … `renderAll()` が state の内容を画面に反映する（状態と見た目を分離）

## カスタマイズのヒント

- **新しいカードを作る** … `CARD_LIBRARY` にエントリを足す（`rarity` を付ければ自動でガチャ排出対象になる。戦闘報酬にも出したいなら `REWARD_POOL` にIDを追加）
- **ガチャの排出率を変える** … `RARITY_TABLE` の `weight` を変更（合計100ならそのまま%）
- **敵を増やす** … `NORMAL_ENEMIES` / `ELITE_ENEMIES` 配列にデータを足すだけでランダム出現する
- **イベントを増やす** … `EVENTS` 配列に「タイトル・説明・選択肢」を足すだけ
- **塔を高くする** … `TOTAL_FLOORS` を変更
- **難易度調整** … プレイヤーの初期HP（`state.player`）や `HEAL_AFTER_BATTLE`、`GACHA_COST` を変更

## 作っていないもの（意図的なスコープ外）

オンライン対戦 / ログイン / データベース保存 / 課金要素 / 既存作品の素材の使用
